# Builder 端产品需求文档 (PRD) - 工程排期版

## 1. 概述
本文档用于 Builder 端（内部工作台）的工程实现与工期评估。涵盖现有的项目/订单管理体系，以及本次新增的「桑基视图」与「项目需求书」功能。

---

## 2. 新增功能模块详细说明 (用于排期评估)

### 2.1 订单管理：桑基视图 (Sankey View)
**需求背景**：在 `/orders` 页面，除了传统的列表展示，新增桑基图视图，用于直观展示项目预决算资金流向各个具体订单的分布情况。

#### 2.1.1 交互与 UI 规范
- **视图切换**：在 `/orders` 页面右上角新增一个 `SegmentedControl`（分段控制器）或 Toggle 按钮，选项为「列表视图」和「资金流向视图（桑基图）」。
- **默认状态**：默认展示「列表视图」，状态可持久化（记录在 localStorage 中）。
- **图表集成**：引入外部做好的桑基图 Demo（基于 ECharts 或 D3.js 等图表库）。
- **交互跳转**：
  - 桑基图的最右侧节点代表具体的子订单（如 `PSO-OD_LHJCF-00014中央空调布管`）。
  - **Hover 态**：鼠标悬浮节点或连线时，显示 Tooltip（包含金额明细、占比等）。
  - **Click 态**：点击最右侧订单节点，触发路由跳转至 `/overview` 页面，并携带该订单的上下文数据（`orderId`）。

#### 2.1.2 数据结构要求 (Mock Data 示例)
前端需要将后端返回的扁平订单数据转换为桑基图所需的 `nodes` 和 `links` 格式：
```json
{
  "nodes": [
    { "name": "预决算" },
    { "name": "MS08-水电工程" },
    { "name": "PSO-OD_LHJCF-00014中央空调布管", "orderId": "o14" }
  ],
  "links": [
    { "source": "预决算", "target": "MS08-水电工程", "value": 195000 },
    { "source": "MS08-水电工程", "target": "PSO-OD_LHJCF-00014中央空调布管", "value": 125000 }
  ]
}
```

#### 2.1.3 验收标准 (Acceptance Criteria)
- [ ] 视图切换平滑，无明显卡顿，切换后保留当前项目的筛选状态。
- [ ] 桑基图渲染正确，节点层级分明（总预算 -> 阶段 -> 具体订单）。
- [ ] 点击最右侧订单节点，能准确跳转到对应的 `/overview` 页面并加载该订单数据。
- [ ] 窗口缩放时，桑基图能够自适应 Resize。

### 2.2 项目管理：项目需求书 (Project Requirements)
**需求背景**：在 `/projects` 页面新增项目需求书功能，支持 Builder 端编辑、发布给 Home 端，并记录完整的修改历史。

#### 2.2.1 交互与 UI 规范
- **入口**：`/projects` 列表的卡片操作区，或项目详情页的 Tab 栏，新增「项目需求书」入口。
- **表单/清单视图**：
  - 读取数据库中的需求字段（如：空间规划、风格偏好、常住人口、预算范围等）。
  - Builder 端可直接在页面上进行行内编辑（Inline Edit）或表单编辑。
- **发布机制 (Publish)**：
  - 页面底部或右上角固定【发布至 Home 端】按钮。
  - 点击发布后，状态变更为「已发布」，Home 端内容同步更新。
  - 发布后**不锁定**编辑，Builder 仍可修改。一旦修改，界面提示“有未发布的更改”，需再次点击【发布】。
- **历史记录 (History Log)**：
  - 提供【查看修改历史】按钮，点击唤出抽屉 (Drawer) 或弹窗 (Modal)。
  - 以时间轴 (Timeline) 组件展示日志。

#### 2.2.2 数据结构要求
**需求书实体 (Requirement)**
```typescript
interface ProjectRequirement {
  projectId: string;
  content: Record<string, any>; // 具体的表单字段
  status: 'draft' | 'published';
  lastPublishedAt: string | null;
}
```
**历史日志实体 (History Log)**
```typescript
interface RequirementHistory {
  id: string;
  projectId: string;
  modifiedAt: string; // 修改时间
  modifiedBy: string; // 修改人姓名
  action: 'edit' | 'publish'; // 动作类型
  changes?: {
    field: string;
    oldValue: any;
    newValue: any;
  }[]; // 具体的字段变更差异
}
```

#### 2.2.3 验收标准 (Acceptance Criteria)
- [ ] 需求书内容能正确从数据库读取并渲染为可编辑表单。
- [ ] 修改内容后，能成功保存到数据库。
- [ ] 点击【发布】后，状态正确更新，且触发记录一条 `action: 'publish'` 的历史日志。
- [ ] 每次保存修改，自动生成一条 `action: 'edit'` 的历史日志，记录修改前后的差异。
- [ ] 历史记录时间轴按时间倒序排列，展示清晰。

## 3. 现有核心模块说明 (用于整体排期参考)

### 3.1 订单概览 (/overview)
- **核心逻辑**：管理设计方案、报价单、结算单的版本。
- **约束条件**：同类型单据全局唯一 `draft`；新建版本必须继承上一版本的图纸、批注和评论。
- **状态流转**：严格遵循 `rules.md` 中的状态机定义。

### 3.2 方案编辑器 (/editor)
- **核心逻辑**：多页图纸展示、打点批注（Annotations）、查看客户评论（Comments）。
- **交互**：图片缩放、拖拽打点、右侧面板联动高亮。

### 3.3 报价单与结算单 (/quotation, /settlement)
- **核心逻辑**：费用明细清单展示，支持 Builder 端编辑金额与数量。
- **状态流转**：未查看 -> 已查看 -> 已反馈/已签字。

## 4. 研发排期建议 (WBS 参考)
1. **基础设施**：路由配置、状态管理、公共组件封装（约 10%）。
2. **桑基图集成**：数据转换逻辑、图表组件封装、点击路由跳转逻辑（约 20%）。
3. **项目需求书**：表单开发、发布逻辑、差异对比算法（用于生成历史记录）、时间轴组件（约 30%）。
4. **现有模块联调**：Overview 版本管理、Editor 打点交互、报价单流转（约 30%）。
5. **测试与修复**：边界条件测试、UI 还原度走查（约 10%）。
