## 2. 方案设计版本状态规则 (Scheme Version Status)
方案状态（Version Status）代表单个设计方案版本的状态。

| 状态代码 | 状态名称 | 描述 | 位置 |
| :--- | :--- | :--- | :--- |
| `draft` | 草稿版本 | 内部创建，未点击发布 | 当前活跃版本区 |
| `published_unread` | 已发布(未读) | 内部点击发布，客户尚未开始审阅 | 当前活跃版本区 / 历史列表 |
| `reviewing` | 客户审阅中 | 客户在终端点击“开始审阅” | 当前活跃版本区 / 历史列表 |
| `reviewed` | 客户已完成审阅 | 客户点击“完成审阅 / 满意并提交” | 当前活跃版本区 / 历史列表 |
| `historical` | 历史版本 | 仅作为状态占位，实际显示时保留原状态名称 | 历史与已发布列表 |

## 3. 方案管理核心逻辑 (Core Logic)

### 3.1 唯一草稿约束 (Unique Draft Constraint)
- 整个订单下**只能存在一个**处于 `draft` 状态的方案。
- 如果当前已存在 `draft`，则**不允许**新建方案。用户必须先发布当前草稿。

### 3.2 版本号递增规则 (Version Numbering)
- 版本号采用纯数字递增：1, 2, 3...
- UI 显示为“内部版本 X”。
- 全局递增，不允许删除版本。
- 每次新建方案时，版本号 = 当前最大版本号 + 1。

### 3.3 方案名称编辑 (Editable Name)
- 仅当方案状态为 `draft` 时允许编辑名称。

### 3.4 复制与继承逻辑 (Copy & Inheritance)
- 新建方案时，自动基于当前订单的**最新版本**生成副本。
- 副本将继承原版本的所有信息，包括：
    - 设计图页面
    - EPC设计注释（标注）
    - 客户反馈参考（评论）
- 新建成功后停留在当前页面，并弹出提示告知用户已继承相关数据。

### 3.5 历史列表显示 (History List Display)
- 被挤入“历史与已发布版本”列表的方案，**保持其原有的状态名称**。
- 历史列表中所有的状态胶囊统一显示为**灰色**。

### 3.6 视图重置 (View Reset)
- 每次进入 `/overview` 页面，视图自动重置到最顶部。
