# 轻量设计规范 v0.1 (Design Spec)

## 1. 设计目标
本规范旨在为当前 Prototype 项目提供统一的视觉基准，确保后续页面迭代在字体、颜色、间距、圆角及核心组件上保持高度一致，降低沟通成本，提升 AI 生成页面的准确度。

## 2. 使用范围
- 适用于本项目所有 Web 页面、模块及组件。
- 适用于前端开发、设计复核以及 AI 提示词参考。

## 3. 字体规范 (Typography)
所有文字必须严格遵守以下层级，禁止随意设置字号与字重。

| 层级 | 字号 (Size) | 字重 (Weight) | 字体族 (Font Family) | 备注 |
| :--- | :--- | :--- | :--- | :--- |
| **页面大标题** | 48px | 900 | 'PingFang SC', 'Microsoft YaHei', sans-serif | 页面唯一 H1 |
| **区块标题** | 30px | 900 | 'PingFang SC', 'Microsoft YaHei', sans-serif | 模块/Section 标题 |
| **正文** | 16px | 400 | HonorSans, sans-serif | 默认阅读文字 |
| **辅助文字** | 12px | 500 | HonorSans, sans-serif | 提示、标注、元数据 |
| **按钮文字** | 16px | 700 | HonorSans, sans-serif | 按钮内部文字 |

## 4. 颜色规范 (Colors)
禁止在规范外引入新的色值。

- **品牌主色**: `#EF6B00` (Primary)
- **主色 Hover**: `#CC5B00` (Primary Hover)
- **页面背景**: `#FFFFFF` (Page Background)
- **卡片背景**: `#FFFFFF` (Card Background)
- **主文字**: `#0A0A0A` (Text Primary)
- **次文字**: `#6B7280` (Text Secondary)
- **边框色**: `#E5E7EB` (Border)
- **辅助蓝色**: `#4887FF` (Secondary Blue) - *仅用于特殊功能或局部强调*
- **链接/强调**: `#2F2D97` (Link/Accent)

## 5. 间距规范 (Spacing)
所有间距必须基于以下原子单位进行组合。

- **基础增量**: `8px`, `12px`, `16px`, `24px`, `32px`
- **页面级间距**:
    - 页面左右内边距 (Padding): `24px`
    - 大区块 (Section) 上下间距: `96px`
- **模块级间距**:
    - 卡片内边距 (Padding): `24px`
    - 小模块/元素间距: `16px`

## 6. 圆角规范 (Border Radius)
- **小圆角**: `12px` (用于输入框、小标签)
- **按钮圆角**: `16px` (标准按钮)
- **卡片圆角**: `24px` (容器、卡片)
- **全圆角**: `999px` (胶囊形状按钮)

## 7. 核心组件规范 (Components)

### 7.1 按钮 (Buttons)
- **主按钮 (Primary)**:
    - 背景: `#EF6B00` / 文字: `#FFFFFF`
    - 字号: `16px` / 字重: `700` / 圆角: `16px`
    - 内边距: `16px 32px`
- **次按钮 (Secondary)**:
    - 背景: `#FFFFFF` / 文字: `#0A0A0A` / 边框: `1px solid #E5E7EB`
    - 圆角: `16px`
    - 内边距: `16px 24px`
- **特殊蓝色按钮**:
    - 仅在需要局部强调或特殊功能时使用 `#4887FF`，不可替代全局主按钮。

### 7.2 卡片 (Cards)
- **背景**: `#FFFFFF`
- **圆角**: `24px`
- **内边距**: `24px`
- **边框**: `1px solid #E5E7EB`
- **阴影**: `0 12px 32px rgba(0, 0, 0, 0.08)`

## 8. 页面容器规范 (Container)
- **页面背景**: 默认统一使用 `#FFFFFF`。
- **布局结构**: 页面左右必须保留 `24px` 的安全边距。
- **节奏感**: 不同功能模块 (Section) 之间必须保持 `96px` 的垂直间距。

## 9. AI 协作使用说明
**后续 AI 在参与本项目开发时，必须严格遵守以下指令：**
1. **规范优先**: 生成新页面或修改组件时，必须优先检索并遵守本规范。
2. **冲突处理**: 若外部参考代码的视觉风格与本规范冲突，必须以本规范为准进行重构。
3. **禁止发散**: 严禁随意引入新的主色、圆角体系或字号。若需新增样式，必须在不破坏现有原子规范（如 8px 倍数间距）的前提下扩展。
4. **结构化属性**: 在编写 CSS 或 Tailwind 类名时，优先使用规范定义的数值。

## 10. 新页面接入检查清单 (Checklist)
- [ ] 字体是否使用了 HonorSans (正文) 和 900 字重 (标题)？
- [ ] 页面背景是否为纯白 `#FFFFFF`？
- [ ] 主按钮是否为橙色 `#EF6B00` 且圆角为 `16px`？
- [ ] 卡片圆角是否统一为 `24px`？
- [ ] 页面左右 Padding 是否为 `24px`？
- [ ] Section 间距是否为 `96px`？
- [ ] 是否引入了规范外的颜色？（应为“否”）
