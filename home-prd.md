# Home 端产品需求文档 (Home PRD) - 工程排期版

## 1. 概述
本文档用于 Home 端（业主/客户端）的工程实现与工期评估。涵盖客户查看项目需求书、审阅设计方案、核对报价单与结算单的核心交互逻辑。

---

## 2. 核心功能模块详细说明 (用于排期评估)

### 2.1 项目需求书确认 (Project Requirements)
**需求背景**：客户在 Home 端查看 Builder 端发布的最新版项目需求书，确认前期的空间规划、风格偏好等信息。

#### 2.1.1 交互与 UI 规范
- **入口**：Home 端首页或项目详情页的显著位置。
- **只读视图 (Read-only View)**：
  - 接收 Builder 端发布的最新版项目需求书数据。
  - 以清晰的卡片或列表形式展示（如：空间规划、材质偏好、常住人口、预算范围等）。
- **版本同步与刷新**：
  - 当 Builder 端修改并重新【发布】后，Home 端自动刷新为最新内容（可通过轮询或 WebSocket，或在页面加载时重新拉取）。
  - 若当前查看的不是最新版本，页面顶部需提示“需求书已更新，请刷新查看”。
- **确认机制 (可选)**：
  - 页面底部提供【确认无误】或【提出修改意见】的按钮。
  - 点击【提出修改意见】弹出输入框，提交后反馈给 Builder 端。

#### 2.1.2 验收标准 (Acceptance Criteria)
- [ ] 需求书内容能正确从数据库读取并渲染为只读视图。
- [ ] Builder 端发布新版本后，Home 端能及时获取并展示最新内容。
- [ ] 客户点击确认或提交修改意见后，状态正确更新并反馈给 Builder 端。

---

### 2.2 方案审阅 (Scheme Review)
**需求背景**：客户对 Builder 端提交的设计图纸进行可视化审阅，并提供反馈意见。

#### 2.2.1 交互与 UI 规范
- **状态流转触发**：
  - 客户收到新方案（状态：`published_unread`）。
  - 客户点击“开始审阅”（状态变为 `reviewing`，Builder 端同步可见）。
- **图纸查看与评论 (Comments)**：
  - 客户可以在设计图纸的任意位置点击打点，弹出输入框添加反馈意见（如：“这里不要挂画”、“沙发颜色换深一点”）。
  - 客户可以查看 Builder 端留下的专业批注（Annotations），但不可修改。
  - 右侧面板展示所有评论和批注的列表，点击列表项可高亮图纸上的对应点。
- **完成审阅**：
  - 客户审阅完毕后，点击页面底部的【完成审阅 / 满意并提交】按钮。
  - 方案状态变为 `reviewed`，该版本在 Home 端锁定，不可再添加评论。

#### 2.2.2 验收标准 (Acceptance Criteria)
- [ ] 客户点击“开始审阅”后，状态正确更新为 `reviewing`。
- [ ] 客户能在图纸上准确打点并添加评论，坐标记录正确。
- [ ] 客户能清晰区分自己的评论和 Builder 端的批注。
- [ ] 客户点击“完成审阅”后，状态正确更新为 `reviewed`，且当前版本锁定不可再评论。

---

### 2.3 报价单与结算单确认 (Quotation & Settlement)
**需求背景**：客户对财务明细的核对与最终确认，并进行电子签名。

#### 2.3.1 交互与 UI 规范
- **状态流转触发**：
  - 客户收到新单据（状态：未查看未签字）。
  - 客户打开详情页（状态变为：已查看未签字）。
- **明细核对**：
  - 清晰展示各项费用明细（设计费、产品费、施工费等）及总价。
- **反馈机制**：
  - 客户若有异议，点击【希望调整方案】并填写意见提交（状态变为：已查看已反馈）。
- **电子签名 (E-Signature)**：
  - 客户确认无误后，点击【确认单据】弹出电子签名画板。
  - 客户在画板上签名并提交。
  - 签名完成后（状态变为：已查看已签字），单据生效，订单状态可能随之流转（如 S02 -> S03）。

#### 2.3.2 验收标准 (Acceptance Criteria)
- [ ] 客户打开单据详情页后，状态正确更新为“已查看未签字”。
- [ ] 费用明细展示准确无误，总价计算正确。
- [ ] 客户提交反馈意见后，状态正确更新为“已查看已反馈”。
- [ ] 电子签名画板交互顺畅，签名图片能成功保存并关联至单据。
- [ ] 签名完成后，状态正确更新为“已查看已签字”。

---

## 3. 消息与通知 (Notifications)
- **通知机制**：当 Builder 端发布新的需求书、设计方案、报价单或结算单时，Home 端应收到相应的消息提醒（如红点、系统通知或短信/微信推送，具体视接入渠道而定）。
- **历史归档**：历史版本（被新版本覆盖的交付物）在 Home 端仅作为只读存档展示，不可再进行评论或签字操作。

---

## 4. 研发排期建议 (WBS 参考)
1. **基础设施**：路由配置、状态管理、公共组件封装（约 15%）。
2. **项目需求书**：只读视图开发、状态同步逻辑、确认/反馈交互（约 20%）。
3. **方案审阅**：图纸查看器集成、打点评论交互、状态流转逻辑（约 35%）。
4. **报价单/结算单**：明细展示、反馈逻辑、电子签名画板集成（约 20%）。
5. **测试与修复**：多端适配测试、UI 还原度走查（约 10%）。
